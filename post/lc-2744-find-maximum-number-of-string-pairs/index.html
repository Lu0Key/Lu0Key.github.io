2744.最大字符串配对数目

给你一个下标从 0 开始的数组 words ，数组中包含 互不相同 的字符串。

如果字符串 words[i] 与字符串 words[j] 满足以下条件，我们称它们可以匹配：

  • 字符串 words[i] 等于 words[j] 的反转字符串.
  • 0 <= i < j < words.length

请你返回数组 words 中的 最大 匹配数目。

注意，每个字符串最多匹配一次。

示例 1：

示例 1：

输入：words = [“cd”,“ac”,“dc”,“ca”,“zz”]
输出：2 解释：在此示例中，我们可以通过以下方式匹配 2 对字符串：

  • 我们将第 0 个字符串与第 2 个字符串匹配，因为 word[0] 的反转字符串是 “dc” 并且等于 words[2]。
  • 我们将第 1 个字符串与第 3 个字符串匹配，因为 word[1] 的反转字符串是 “ca” 并且等于 words[3]。

可以证明最多匹配数目是 2 。

示例 2:

输入：words = [“ab”,“ba”,“cc”]
输出：1 解释：在此示例中，我们可以通过以下方式匹配 1 对字符串：

  • 我们将第 0 个字符串与第 1 个字符串匹配，因为 words[1] 的反转字符串 “ab” 与 words[0] 相等。

可以证明最多匹配数目是 1 。

示例 3:

输入：words = [“aa”,“ab”] 输出：0
解释：这个例子中，无法匹配任何字符串。

提示：

  • 1 <= words.length <= 50
  • words[i].length == 2
  • words 包含的字符串互不相同。
  • words[i] 只包含小写英文字母。

解答

今天这题是简单难度，因为题目说了字符串互不相同，因此每个字符串都是唯一的，不会有一个字符串配对两个的情况。最简单的方法就是一个个过，记 n=words.length, 对于 word[i], 我们从 words[i+1] 到 words[n-1] 中找有没有满足条件的字符串。或者干脆每次都是全部遍历一遍，这样最后的结果除以2就行。

毕竟是学习算法，这里试图用 O(n) 的时间复杂度去解决问题. 就想到，用一个HashMap去存，但是因为每个字符串只出现一次，因此不需要记录字符串出现的次数，因此可以把HashMap 换成 HashSet, 如果这个字符串在HashSet出现过，那么就结果加一，如果没出现过就把字符串的反转放在 HashSet 里. 这样只要一轮就能得到结果。

以下是这个想法的实现

class Solution {
    public StringBuilder sb = new StringBuilder();
    public int maximumNumberOfStringPairs(String[] words) {
        int result = 0;
        Set<String> set = new HashSet<>();

        for(int i =0;i< words.length;i++){
            if(set.contains(words[i])){
                result++;
            }else{
                set.add(reverseS(words[i]));
            }
        }
        return result;
    }

    public String reverseS(String a){
        sb.setLength(0);
        sb.append(a.charAt(1)).append(a.charAt(0));
        return sb.toString();
    }
}
